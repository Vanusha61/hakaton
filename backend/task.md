# Реализовать следующие эндпониты

## Получение количества просмотренных видео по четверти и году:

Кол-во просмотренных видео занятий пользователем относительно количества всех видеозанятий (Важно учесть, что вебинары становятся записями после того, как трансляция закончится, а значит ненужно 2 раза считать вебинар в трансляции и в записи)

### Для четверти

структура ответа
```json
{
  "course_id": <int>,
  "user_id": <int>,
  "quarter": <int>, # 1, 2, 3 or 4
  "watched_count": <int>, # количество просмотренных занятий
  "total_count": <int>,  # всего видео занятий
}
```

### Для года

структура ответа
```json
{
  "course_id": <int>,
  "user_id": <int>,
  "watched_count": <int>, # количество просмотренных занятий
  "total_count": <int>,  # всего видео занятий
}
```

## Полученные баллы за четверть и год
Баллы полученные пользователем относительно макимального кол-ва баллов, которые можно получить

### Для четверти

структура ответа
```json
{
  "course_id": <int>,
  "user_id": <int>,
  "quarter": <int>, # 1, 2, 3 or 4
  "earned_count": <int>, # количество полученных баллов
  "total_count": <int>,  # всего количество баллов за четверть
}
```

### Для года

структура ответа
```json
{
  "course_id": <int>,
  "user_id": <int>,
  "earned_count": <int>, # количество полученных баллов
  "total_count": <int>,  # всего количество баллов за год
}
```

## Изученные конспекты за четверть и год
Кол-во изученных (открытых) пользователем конспектов и вспомогательных материалов относительно всего количества материалов 15/25

### Для четверти

```json
  "total_count": <int>,  # всего конспектов за четверть
}
```

### Для года

структура ответа
```json
{
  "course_id": <int>,
  "user_id": <int>,
```
